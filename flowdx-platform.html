FlowDX Platform

Plataforma Interna de Desenvolvedor (IDP) focada em self-service para provisionamento de infraestrutura e entrega de aplicações.

## Overview

FlowDX Platform é uma iniciativa de plataforma interna que visa simplificar e padronizar o ciclo de vida de aplicações, desde o provisionamento de infraestrutura até a entrega contínua em produção.

O projeto nasce da necessidade de reduzir a fricção entre equipes de desenvolvimento e operações, oferecendo abstrações claras e workflows automatizados que permitem que desenvolvedores foquem no que importa: entregar valor através de software.

A plataforma é construída sobre princípios de GitOps, Infrastructure as Code e observabilidade, garantindo que cada entrega seja rastreável, reproduzível e auditável.

## Evolution / Roadmap

A evolução da plataforma segue um caminho incremental, onde cada fase entrega valor tangível enquanto constrói a base para funcionalidades futuras.

PHASE 1 — CURRENT

Foundation & Reference Workload

Estabelecimento da arquitetura base, workload de referência funcional e pipelines CI/CD automatizados.

PHASE 2 — PLANNED

CLI & Project Templates

Desenvolvimento de CLI para scaffolding de projetos e templates padronizados para diferentes tipos de workload.

PHASE 3 — FUTURE

Delivery Engine & Self-Service Portal

Motor de entrega automatizada e portal web para self-service completo de infraestrutura e deployments.

## FlowDX Components

A plataforma é composta por módulos independentes que podem ser evoluídos e versionados de forma isolada.

⬡ flowdx-app

Aplicação de referência que serve como workload de validação para toda a plataforma.

⬡ flowdx-infra

Módulos Terraform reutilizáveis para provisionamento de infraestrutura AWS.

⬡ flowdx-pipelines

Workflows GitHub Actions padronizados para CI/CD e automação de deploy.

⬡ flowdx-cli

Ferramenta de linha de comando para scaffolding e gerenciamento de projetos.

## Purpose

FlowDX Platform existe para resolver problemas reais de entrega de software, ao mesmo tempo que serve como um laboratório prático para explorar e aplicar conceitos modernos de engenharia de plataforma.

01

Reduzir Fricção

Eliminar barreiras entre desenvolvimento e operações através de abstrações claras e workflows automatizados.

02

Padronizar Entregas

Criar padrões reutilizáveis que garantam consistência e qualidade em todos os deployments.

03

Acelerar Ciclo de Entrega

Permitir que equipes entreguem software mais rápido com confiança e segurança.

04

Aprendizado Contínuo

Servir como plataforma de experimentação para tecnologias e práticas modernas de engenharia.
